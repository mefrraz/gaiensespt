# Plano Final: Client-Side FPB API (Zero GitHub Actions Automáticos)

## Resumo da Arquitetura

**Objetivo:** Apenas client-side calls. Dados atualizados SEM GitHub Actions automáticos. Cache de 5 minutos.

### Fluxo Final
```
User visita site
     ↓
Check Supabase: updated_at > 5 min atrás?
     ↓
SIM → Mostra dados do Supabase imediatamente
     ↓
Em background: fetch da API FPB + update Supabase
     ↓
NO → Mostra loading + fetch da API FPB + update Supabase
```

## API FPB - Detalhes Técnicos

### Endpoint Principal: `get_more_days`

**URL:** `https://www.fpb.pt/wp-admin/admin-ajax.php`
**Method:** GET
**Response:** JSON com campo `result` contendo HTML renderizado

#### Parâmetros Suportados:

| Parâmetro | Tipo | Obrigatório | Exemplo |
|-----------|------|-------------|---------|
| `action` | string | ✅ | `get_more_days` |
| `epoca` | string | ✅ | `2025/2026` |
| `escalao` | string | ✅ | `Sénior` |
| `genero` | string | ✅ | `masculino` ou `feminino` |
| `competicao[]` | array | ⚠️ | `10904` (1ª Divisão Masculina) |
| `clube` | integer | ❌ | `119` (FC Gaia) |
| `period[time_option]` | string | ❌ | `fromInit` ou `RECENTES` |
| `period[from_date]` | string | ❌ | `2025/09/01` |
| `period[to_date]` | string | ❌ | `2026/07/31` |

#### Exemplo de Request GET:
```
GET /wp-admin/admin-ajax.php?action=get_more_days&epoca=2025/2026&escalao=Sénior&genero=masculino&competicao[]=10904&clube=119&period[time_option]=fromInit&period[from_date]=2025/09/01&period[to_date]=2026/07/31
```

#### Response Structure:
```json
{
  "result": "<HTML string com blocos .day-wrapper>",
  "hasmore": true
}
```

## Como a API é Usada - Side Client vs Backend

### Client-Side (React Browser)

#### Onde: `web/src/lib/fpbApi.ts`
- **Quem chama:** React hooks nos componentes
- **Como:** `fetch()` direto do browser
- **CORS:** ✅ Funciona (WordPress não bloqueia)
- **User-Agent:** Navegador padrão

#### Fluxo:
```
Browser → fetch(fpb.pt/admin-ajax.php) → HTML response → DOMParser → JSON array → Supabase upsert
```

#### Vantagens:
- ✅ Zero latência de proxy
- ✅ Menor custo de execução
- ✅ User vê dados frescos

#### Limitações:
- ❌ Cada user faz o request
- ❌ Parsing no browser (mais lento)

### Backend (API Routes - se necessário)

#### Onde: `web/api/fpb-proxy.ts` (opcional)
- **Quando usar:** Se CORS falhar
- **Como funciona:** 
```typescript
// api/fpb-proxy.ts
export default async function handler(req, res) {
  const response = await fetch(`https://www.fpb.pt/wp-admin/admin-ajax.php?${new URLSearchParams(req.query)}`);
  const data = await response.json();
  res.status(200).json(data);
}
```

#### Vantagens:
- ✅ Cache no Vercel edge
- ✅ Parsing no servidor
- ✅ Retry automático

#### Desvantagens:
- ❌ Adiciona complexidade
- ❌ Custo de serverless

### 3. Client-Side (React) - NOVOS ARQUIVOS

#### `web/src/lib/fpbApi.ts`
```typescript
const FPB_BASE = "https://www.fpb.pt/wp-admin/admin-ajax.php";

export async function fetchFPBGames(epoca: string, clube?: number) {
  const searchParams = new URLSearchParams({
    action: "get_more_days",
    epoca,
    escalao: "Sénior",
    genero: "masculino",
    "period[time_option]": "fromInit",
    "period[from_date]": "2025/09/01",
    "period[to_date]": "2026/07/31"
  });
  
  if (clube) searchParams.append("clube", clube.toString());
  
  const res = await fetch(`${FPB_BASE}?${searchParams}`);
  const json = await res.json();
  return parseGamesHTML(json.result);
}

function parseGamesHTML(html: string): Game[] {
  const parser = new DOMParser();
  const doc = parser.parseFromString(html, 'text/html');
  
  return Array.from(doc.querySelectorAll('.jogo')).map(el => ({
    id: el.getAttribute('href')?.match(/internalID=(\d+)/)?.[1] || '',
    data: el.closest('.day-wrapper')?.querySelector('h3.date')?.textContent || '',
    equipa_casa: el.querySelector('.team.home .name')?.textContent || '',
    equipa_fora: el.querySelector('.team.away .name')?.textContent || '',
    resultado_casa: parseInt(el.querySelector('.team.home .score')?.textContent || '0'),
    resultado_fora: parseInt(el.querySelector('.team.away .score')?.textContent || '0'),
    status: el.querySelector('.state')?.textContent?.includes('Terminado') ? 'FINALIZADO' : 'AGENDADO'
  }));
}
```

#### `web/src/hooks/useGames.ts`
```typescript
export function useGames(season = "2025/2026", clube = 119) {
  const [games, setGames] = useState<Game[]>([]);
  const [loading, setLoading] = useState(true);
  const [lastUpdated, setLastUpdated] = useState<Date | null>(null);
  
  const tableName = `games_${season.replace('/', '_')}`;
  const CACHE_MINUTES = 5;
  
  useEffect(() => {
    const loadData = async () => {
      // 1. Check cache
      const { data: cached } = await supabase
        .from(tableName)
        .select('*')
        .order('data')
        .limit(1)
        .single();
      
      if (cached) {
        const updatedAt = new Date(cached.updated_at);
        const fiveMinAgo = new Date(Date.now() - CACHE_MINUTES * 60000);
        
        if (updatedAt > fiveMinAgo) {
          // Cache válido - carrega imediato
          const { data } = await supabase.from(tableName).select('*').order('data');
          setGames(data || []);
          setLastUpdated(updatedAt);
          setLoading(false);
        }
      }
      
      // 2. Fetch fresh data (mesmo que tenha cache)
      try {
        const freshData = await fetchFPBGames(season, clube);
        setGames(freshData);
        setLastUpdated(new Date());
        
        // 3. Update Supabase
        await supabase.from(tableName).upsert(
          freshData.map(g => ({ ...g, updated_at: new Date(), id: g.id }))
        );
      } catch (error) {
        console.error('Failed to fetch games:', error);
      } finally {
        setLoading(false);
      }
    };
    
    loadData();
  }, [season, clube]);
  
  return { games, loading, lastUpdated };
}
```
