## 2. Design System — Mobile-First (novo)

### Paleta de Cores
```
Atual:                Novo:
gaia-yellow (#f5b417) → manter (identidade FC Gaia)
zinc-900 → zinc-950   (preto mais profundo)
zinc-100 → zinc-50    (branco mais puro)
Adicionar:
  gaia-black  (#1a1a2e)  → fundos dark
  gaia-green  (#22c55e)  → vitórias
  gaia-red    (#ef4444)  → derrotas
```

### Tipografia
```
Atual: Outfit 300/400/500/600/700
Problema: font-weight 300 é demasiado leve para mobile
Novo: Outfit 400/500/600/700/800 — REMOVER font-weight 300 e 400 de títulos
```

### Espaçamento
```
Atual: Mistura px-2, px-4, p-4 inconsistente
Novo: Sistema consistente:
  card padding: p-4 (mobile), p-6 (tablet+)
  page padding: px-3 (mobile), px-6 (tablet+)
  gap entre secções: space-y-6 → space-y-8
```

### Componentes Base
```
Buttons:
  Atual: classes inline com tailwind
  Novo: Componente <Button> com variantes primary/secondary/ghost

Cards:
  Atual: glass-card com classes repetidas
  Novo: Componente <GameCard> e <DashboardCard>

Segment:
  Atual: inline no JSX com classes condicionais
  Novo: Componente <SegmentControl> com options array
```

## 7. README.md

```
# gaiensespt — FC Gaia Basquetebol

## Stack
React 18 + Vite + TypeScript + Tailwind CSS + Supabase + Vercel

## Fluxo de Dados
Browser → Vercel Edge Function → fpb.pt (página HTML) → parser DOMParser → Supabase (cache)

1. git clone
2. cd web && npm install
3. cp .env.example .env  (preencher SUPABASE_URL, SUPABASE_ANON_KEY)

- Frontend: Vercel (auto-deploy via GitHub)
- Database: Supabase
- API Proxy: Vercel Edge Function em web/api/fpb.ts
```
